VM Start-up Shutdown Time Prediction

About INSTANCE

Elasticity or scalability is the biggest advantage Cloud offers to its users. Based on the user workload, the desired number of virtual machine instances can be acquired on-demand. This on-demand scalability feature has served as the major attraction point for many industries to join the cloud community to help them handle the dynamical workload and reduce their cost. As the user workload increases Cloud Service Providers (CSPs) dynamically add virtual machines though it may take some time for the acquired VMs to be ready to use. It takes time to find a spot to provision a VM in CSPs data centers and further allocate resources (e.g. IP addresses, attach storage) to the VM and to copy, boot and configure the OS image. This whole time-period is called as VM booting time.

This delayed start of the VMs could impact the performance of the cloud user’s applications. Therefore, the cloud users should somehow know how much time it will take to start VMs so that they can plan ahead and make better resource provisioning decisions. This information is particularly important to the time-critical applications that rely on the cloud’s on-demand resources and is dependent on the cloud autoscaling process to make correct decisions.

Analogous to VM booting time, there is some time delay for deallocation of the VM. This time-period is called as VM shutting-down time. As most of the users use Cloud shared resources, therefore knowing this time-period by CSPs could help them to make better decisions of efficiently scheduling and allocation of VMs for the other users.

So here we present INSTANCE, a Go based web application which predicts the booting and shutting-down time for various types of Virtual Machines on different Cloud Service Providers.
